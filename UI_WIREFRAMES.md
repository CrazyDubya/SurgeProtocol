# 1. DESIGN PHILOSOPHY

## 1.1 Core Principles

```
SURGE PROTOCOL UI PRINCIPLES

1. TEXT FIRST
   - Narrative drives experience
   - UI supports, never dominates
   - Every screen tells a story

2. INFORMATION DENSITY
   - Show what matters
   - Hide what doesn't
   - Reveal on demand

3. ATMOSPHERE ALWAYS
   - Cyberpunk aesthetic in typography
   - Ambient details in every view
   - The Algorithm is always watching

4. MOBILE-FRIENDLY
   - Works on phone screens
   - Touch-optimized interactions
   - Responsive layouts

5. ASYNC-AWARE
   - Clear state indicators
   - Time pressure visible
   - Actions feel consequential
```

# 9. MOBILE OPTIMIZATIONS

## 9.1 Key Mobile Differences

```
MOBILE ADAPTATIONS

1. BOTTOM NAVIGATION
   - Fixed tab bar at bottom
   - 5 main sections: Map, Missions, Character, Inventory, Factions
   - FAB for active mission when applicable

2. CARD-BASED LAYOUTS
   - Swipeable mission cards
   - Collapsible sections
   - Pull-to-refresh

3. COMBAT
   - Simplified grid (tap to select squares)
   - Action buttons larger
   - Enemy cards swipeable
   - Portrait orientation prioritized

4. ALGORITHM
   - Chat-style interface
   - Quick reply chips
   - Notification-style ambient messages

5. GESTURES
   - Swipe left: dismiss/decline
   - Swipe right: accept/confirm
   - Long press: details/context menu
   - Pull down: refresh
```
